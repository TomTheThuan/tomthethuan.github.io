    Binary #

    Computers read and interpret data differently. Computers only understand data in a binary form, which is 1s and 0s.

    Computers anr simply a collection of billions of switches which can be either on (represent by the digit 1) and off (represent by the digit 0).

    In binary we have only two digits which are 1 and 0, so we can call this number system ‘base-2’.

    A bit (short for ‘Binary Digit’) is the smallest unit of data in a computer and has a single binary value of 1 or 0.

    Units in binary: # 

    1 bit = a single 0 or 1
1 byte = 8 bits = 1 character of text = 2 nibbles
1 KB = 1024 bytes
1 MB = 1024 KB or 1024 x 1024 bytes
1 GB = 1024 MB
1 TB = 1024 GB

    A sence of scale

    File Size
    One character of text 1 byte
    A full page of text 30 KB
    One small digital colour photograph 3 MB
    Music CD 600 MB
    A DVD 4.5 GB
    Hard disk 1 TB

    Understanding Denary / Decimal system #

    For humans to be able to work with computers we need ways of representing all of the data we use in binary.

    Computers use a binary number system consisting of only 0s and 1s.

    Everything that a computer needs to process must beconverted into a binary format.

    This format is used for storing numbers, text, images, sound and program instructions.

    Humans use natural numbers daily and can be described as denary, decimal or base-10 system of numbers.

    How the number system works #

    The example of 2703 in denary:

    $2703_{10} = 21000 + 7100 + 010 + 31$

    In this example, digit 2 represents there are 2 thousands, digit 7 represents there are 7 hundreds, no tens because the digit on tens is 0, and 3 ones.

    $$ 2703_{10} = 21000 + 7100 + 010 + 31 = 210^3 + 710^2 + 010^1 + 310^0 $$

    And also, binary works in the same way, but change the base.

    $$ 00111001_2 = 02^7 + 02^6 + 12^5 + 12^4 + 12^3 + 02^2 + 02^1 + 02^0 = 57_{10} $$

    Convert binary to denary #

    $$ 00111001_2 = 02^7 + 02^6 + 12^5 + 12^4 + 12^3 + 02^2 + 02^1 + 02^0 = 57_{10} $$

    All the digits time their place value then add them together.

    Hexadecimal System #

    Understanding Hexadecimal #

    Even throught all data inside a computer is manipulated using binary code. If we should ever need to document binary code outside of the system. It is helpful to understand the internal code.

    A Hexadecimal system is a number system that is more effective for documentation purposes.

    Hexadecimal number are in base-16 system and therefore uses 16 symbols (numbers and characters) to represent numbers.

    Base-16 use number 0~9 and letters A~F

    A - 10
B - 11
C - 12
D - 13
E - 14
F - 15

    Why are we using Hex? #

    A hexadecimal value is much easier to read and remember than a string of binary digits.

    It is quicker to write or type, since a hex digit takes up only one character, not four digits in binary.

    There is less chance of makin an error when typing hex characers than a string of 1s and0s.

    It is easy to convert to and from binary.

    Hexadecimal in HTML colours #

    Colours are defining in HTML using the hexadecimal format and are represented using 6 digits or symbols (characters or numbers).

    $$ #FFFFFF $$

    • Frist two digits represent the amount of red in the colour (in he range of 00 to FF, or 000 to 255 in decimal)

    • Next two digits represent the amount of gree in he colour.

    • Final two digits represent the amount of blue in the colour

    By changing the intensity (amount) of red, green and blue, we can create almost any colour.

    In the colour $#00FF80$, there are no red in this this colour, $FF$ green and $80$ blue.

    Hexadecimal in MAC address #

    The Media Access Control (MAC) somtimes called hardware address, is a 12 digit number used to uniquely identity computers or devices.

    These numbers are physically burned into the netbook hardware during the manufacturing process, or stored in firmware, and design to not be modified. They are usually written in hexadecimal format.

    An example of a MAC address would be 00-A3-63-84-45-E6.

    The first 6 digits identity the manufacturer while the others give the serial number or device ID.

    Hexadecimal in Machine Code and Assemble Language #

    Machine code Assemble codes are ow-level programming language. This means thay can be used quickly by a computer. They are languages that computers can interpret and understand.

    Assemble code used hexadecimal.

    Using hexadecimal makes it much easier, faster and more error-free to write code compared to machine code.

    Using machine code is very difficult and it takes a long time to type in values. It is also very easy to mistype the digits in a ‘sea of 0s and 1s’.
